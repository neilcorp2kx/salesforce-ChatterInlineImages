# v1.0.0 — Initial release

Paste screenshots directly into Chatter posts and have them render inline in Lightning Experience.

A working solution to an [IdeaExchange request with 9,450 points](https://ideas.salesforce.com/s/idea/a0B8W00000PFdoSUAT/enable-pasting-images-in-chatter-for-lightning-experience) that Salesforce marked **"Archived: Never"** — without a browser extension, without `Click link for image` fallback text, without hand-rolled rta-image workarounds.

![Demo](https://github.com/neilcorp2kx/salesforce-ChatterInlineImages/raw/main/ChatterInlineImagesSample.gif)

## What's included

- **`chatterImageEditor`** — Flow screen LWC with document-level paste capture and focus tracking
- **`ChatterInlineImagesController`** — Apex for uploading pasted images as `ContentVersion` records
- **`ChatterInlineImagePoster`** — `@InvocableMethod` that parses rich text HTML and posts via `ConnectApi.ChatterFeeds.postFeedElement` with inline image and mention segments in document order
- **`Chatter_Inline_Image_Post_Demo`** — Minimal 2-element demo flow (Screen → Post to Chatter)
- **`Post_Inline_Comment`** — Case quick action that launches the demo flow
- **25 Apex tests** covering parsing logic, invocable error paths, and a full ConnectApi end-to-end post with a real inline image

## Features

- Paste a screenshot → uploads as a real Salesforce File → renders inline in the native Chatter feed
- `@mention` users with real `MentionSegment` (bell notifications included)
- Visibility selector (`Internal Users Only` / `All with Access`)
- Object-agnostic — works on any parent record type via `{!recordId}` from flow context
- Compatible with Salesforce API v62.0

## Install

```bash
git clone --branch v1.0.0 https://github.com/neilcorp2kx/salesforce-ChatterInlineImages.git
cd salesforce-ChatterInlineImages
sf project deploy start --manifest manifest/package.xml --target-org <your-org-alias>
```

Then add the **Post Inline Comment** quick action to your Case page layout. Full setup and adaptation instructions in the [README](https://github.com/neilcorp2kx/salesforce-ChatterInlineImages#install).

## The trick

Attach a `paste` listener to `document` in the capture phase, gate it on a focus flag tracked via `focusin`/`focusout` on the LWC host, and `preventDefault()` before `lightning-input-rich-text` ever sees the event — then upload the image bytes as a real `ContentVersion` and emit `ConnectApi.InlineImageSegmentInput` segments when the post is submitted.

The README walks through the five dead ends you'll hit trying to do this any other way (rta-image interception, ConnectApi requirements, Quill sanitization, shadow DOM event targeting, Lightning Web Security path retargeting) and why this is the one path that actually works.

## Known limitations

- Pasted images append at the end of the editor, not at cursor position
- Mention picker is a button, not inline `@` typeahead
- No drag-drop support yet (paste only)
- If Salesforce ever changes how `lightning-input-rich-text` dispatches paste events, this could break
